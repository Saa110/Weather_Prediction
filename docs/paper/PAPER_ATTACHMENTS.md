# Paper Attachments: Tables and Figures

Use these files when submitting or attaching results to your paper. All are generated by `scripts/research/generate_paper_assets.py`.

**Recommended toolchain:** **pandas + matplotlib + seaborn** (already in your environment). No separate “dashboard” tool — we export static tables and figures to files you can attach or include in LaTeX/Word.

---

## Output Directories

| Directory | Contents |
|-----------|----------|
| `docs/paper/tables/` | CSV and LaTeX tables |
| `docs/paper/figures/` | PDF (vector) and PNG (raster) figures |

## Summary Checklist

**Main text:**
- [ ] Table 1 (stations)
- [ ] Table 2 (data source)
- [ ] Table 3 (architecture)
- [ ] Table 4 (window MaxT)
- [ ] Table 5 (window MinT)
- [ ] Table 6 (baseline comparison)
- [ ] Table 7 (feature importance)
- [ ] Table 8 (ablation)
- [ ] Table 9 (seasonal)
- [ ] Figure 1 – Figure 7 (and optionally Figure 8)
- [ ] Figure 9 (ablation)
- [ ] Figure 10 / 10b (seasonal)

**Supplementary:**
- [ ] Table S1: Per-station MAE (`table_supp_per_station_mae.csv`)

**Formats:**
- **Tables:** Use CSV for Excel/Word; use `.tex` for LaTeX.
- **Figures:** Use **PDF** for LaTeX (vector); use **PNG** (e.g. 300 dpi) for Word or submission portals that require raster.

## How to Generate

From the project root:

```bash
python scripts/research/generate_paper_assets.py
```

This reads `data/research/results.db`, `docs/analysis/feature_importance_raw.csv`, and (for ablation) `data/research/results.db` table `ablation_runs` or `docs/analysis/ablation_results.csv`, then writes all tables and figures above into `docs/paper/tables/` and `docs/paper/figures/`.
